# Support

Inline Diff is distributed through the VS Code Marketplace and can also be
validated locally from a packaged VSIX.

## Reporting Issues

When reporting a problem, include:

- VS Code or compatible editor name and version
- Operating system and Git version
- Inline Diff version
- Whether the workspace was newly initialized or reused an existing
  `.inlinediff/` directory
- Steps to reproduce the issue
- What you expected to happen
- What actually happened

Do not attach private source files or the contents of `.inlinediff/` unless you
have reviewed them and are comfortable sharing them publicly.

## Safety-Critical Reports

For reports involving lost changes, corrupted files, incorrect Reject results,
or unexpected baseline changes, include the timeline of actions as precisely as
possible. Note whether another editor, formatter, AI coding tool, build tool,
or script was modifying the same files at the same time.
